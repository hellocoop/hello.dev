# OpenID Config

Many platforms have a generic OpenID Connect (OIDC) plug-in / module / configuration. To use Hellō with these platforms you will need to copy and paste configuration information between Hellō and the platform. We suggest having browser windows open with both the Hellō console and the platform administrative console.


<Steps>

### Hellō Configuration
The platform will need the Hellō configuration. Some platforms will retrieve the configuration from the Hellō [`openid-configuration`](https://issuer.hello.coop/.well-known/openid-configuration) file, or you may need to enter the values in: 

| Parameter                | URL                                                                                                                  |
|--------------------------|----------------------------------------------------------------------------------------------------------------------|
| `openid-configuration`   | <Pre hasCopyCode={true} style={{marginBottom: 0}}>`https://issuer.hello.coop/.well-known/openid-configuration`</Pre> |
| `issuer`                 | <Pre hasCopyCode={true} style={{marginBottom: 0}}>`https://issuer.hello.coop`</Pre>                                  |
| `authorization_endpoint` | <Pre hasCopyCode={true} style={{marginBottom: 0}}>`https://wallet.hello.coop/authorize`</Pre>                        |
| `token_endpoint`         | <Pre hasCopyCode={true} style={{marginBottom: 0}}>`https://wallet.hello.coop/oauth/token`</Pre>                      |
| `userinfo_endpoint`      | <Pre hasCopyCode={true} style={{marginBottom: 0}}>`https://wallet.hello.coop/oauth/userinfo`</Pre>                   |

### Client ID
The platform will need the client id from the Hellō console. 

### Client Secret
Most platforms will require a client secret. You can generate one or more in the [Hellō console](https://console.hello.coop).

### Redirect URI
Hellō will need the redirect URI from the platform. This is where Hellō will redirect the user after interacting at Hellō.

### Scopes 
The platform will have a place to specify the scopes that you would like to request from the user. See [Hellō Claims](/docs/scopes) for supported scopes.

### Hellō Buttons
There will be some mechanism for the Hellō button to initiate the authentication flow.

</Steps>
